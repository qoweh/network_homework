# 특성화 계획 : 기능 추가

### 기능 1: 암호화 통신

**기능 설명:**

- 채팅 메시지를 암호화하여 전송
- 간단한 XOR 암호화 또는 AES-128 사용
- 헤더에 암호화 플래그(1bit) 추가

**패킷 헤더 변경:**

```
기존: [Type(1B)] [Seq(4B)] [Total(4B)] [Data]
추가: [Type+Flag(1B)] [Seq(4B)] [Total(4B)] [Data(암호화됨)]
      └─ 최상위 비트가 1이면 암호화됨
```

**방법:**

- GUI에 "암호화 전송" 체크박스 추가
- Wireshark로 패킷 캡처하여 평문 vs 암호문 비교

### 기능 2: 우선순위(Priority) 큐

**기능 설명:**

- 메시지에 우선순위 부여 (긴급/일반/낮음)
- 우선순위 높은 메시지 먼저 처리
- IP 헤더의 TOS(Type of Service) 필드 활용

**패킷 헤더 변경:**

```
IP 헤더의 TOS 필드 활용:
[Ver(4bit)][IHL(4bit)][TOS(8bit)]...
                        └─ 우선순위 (상위 3비트)
```

**방법:**

- 낮은 우선순위 메시지 10개 전송 → 긴급 메시지 1개 전송
- 긴급 메시지가 먼저 표시됨을 보여줌

### 기능 3: 타임스탬프 및 로깅

**기능 설명:**

- 모든 메시지에 타임스탬프 추가
- 전송/수신 시간 측정 및 표시
- 패킷 로그를 파일로 저장

**패킷 헤더 변경:**

```
기존: [Type(1B)] [Data]
추가: [Type(1B)] [Timestamp(8B)] [Data]
```

**방법:**

- 각 메시지의 전송 시간과 지연 시간 표시
- 로그 파일(`packet.log`)을 열어서 모든 통신 기록 확인
